# eBook Listings - Kindle出版メタデータ一括生成スキル

電子書籍の原稿テキスト → キーワードリサーチ → タイトル・著者名・キーワード7つ・紹介文を一括生成。

## When to Use This Skill

- 「紹介文を作って」「Kindleの紹介文」
- 「タイトル案を出して」「タイトルとキーワードを作って」
- 「電子書籍のメタデータ」「Kindle出版用の情報」
- 「ebook-listings」
- 原稿テキストを渡されて出版情報の作成を求められた場合

## 全体フロー（3フェーズ）

```
Phase 1: 原稿分析
   │  原稿テキストからテーマ・ターゲット・キーコンセプトを抽出
   │  ユーザー指定のキーワードがあれば取得
   ▼
Phase 2: キーワードリサーチ
   │  WebSearchでAmazon/Kindle市場のキーワード調査
   │  競合書籍の傾向分析 → キーワード候補15-20個 → 最適な7つを選定
   ▼
Phase 3: メタデータ生成
   │  タイトル3案 + 著者名 + キーワード7つ + 紹介文
   ▼
完成！ → output/ebook-listings-{slug}/metadata.md に出力
```

## 生成物の仕様

| 項目 | 内容 |
|------|------|
| タイトル・副タイトル | 3案（キーワード密度重視） |
| 著者名 | 「〇〇研究室」形式（キーワード入り） |
| キーワード | 7つ（Kindle登録用） |
| 紹介文 | Kindle商品ページ用（Kindle制約準拠） |
| 出力形式 | Markdown（metadata.md） |

## 出力先

```
output/ebook-listings-{テーマslug}/
└── metadata.md    # タイトル案・著者名・KW7つ・紹介文すべてを含む
```

---

## Phase 1: 原稿分析

ユーザーから渡された原稿テキストを読み、以下を抽出する：

1. **メインテーマ**: 本の中心的なトピック
2. **ターゲット読者**: 誰に向けた本か（年齢層、職業、悩み）
3. **キーコンセプト**: 本が提供する価値・解決策のコア
4. **本の特長**: マンガ付き、図解数、ページ数など差別化要素
5. **目次**: 章立て・見出し構成
6. **トーン**: おだやか系 / ガツガツ系 / 技術入門系 / 戦略系
7. **ユーザー指定キーワード**: 原稿内で既に指定されているキーワード

---

## Phase 2: キーワードリサーチ

### リサーチ手順

1. **シードキーワード抽出**: Phase 1の分析結果からシードキーワードを3-5個抽出
2. **WebSearch実行**（最大3回）:
   - 検索1: `{メインテーマ} Kindle 本 Amazon`
   - 検索2: `{メインテーマ} 電子書籍 おすすめ`
   - 検索3: `{関連テーマ} Kindle ベストセラー`
3. **競合分析**: 検索結果から競合書籍のタイトル・キーワード傾向を把握
4. **キーワード候補リスト作成**: 15-20個のキーワード候補を生成
5. **最終選定**: 以下の基準で7つに絞り込む

### キーワード選定基準

| 優先度 | 基準 |
|--------|------|
| 高 | 検索ボリュームが大きい（一般的な用語） |
| 高 | 本の内容と直接関連する |
| 中 | ロングテール（3語以上の複合キーワード） |
| 中 | 競合が使用しているキーワード |
| 低 | ニッチだが購買意図が高い |

### キーワード構成の目安

- メインキーワード: 2-3個（テーマの核心）
- 関連キーワード: 2-3個（周辺トピック・類義語）
- ロングテール/ニッチ: 1-2個（差別化）

---

## Phase 3: メタデータ生成

### 3-1. タイトル・副タイトル（3案）

各提案は以下の構造で作成する：

```
【提案N：コンセプトの一言説明】
タイトル
{キーワードを含む、30文字以内のインパクトあるタイトル}

副タイトル
{キーワードを含む、本の価値と読者ベネフィットを伝える副タイトル}
```

**タイトル作成ルール:**
- 必ず1つ以上のメインキーワードを含める
- 30文字以内を目安（長すぎると検索で切れる）
- 「」で重要キーワードを囲むと目立つ
- 数字を入れると具体性が増す

**副タイトル作成ルール:**
- 追加のキーワードを自然に盛り込む
- 読者が「自分のための本だ」と感じるベネフィットを記述
- 具体的な成果・数字を含めると効果的

**3案の方向性:**
- 提案1: 網羅性・キーワード密度重視
- 提案2: 好奇心・秘匿性で引きつける
- 提案3: 簡単さ・具体的成果を打ち出す

### 3-2. 著者名

```
{テーマキーワード}+{ジャンル修飾語}+研究室
```

**作成ルール:**
- Amazon検索でヒットするキーワードを含める
- 「研究室」「ラボ」「アカデミー」等の信頼感のある語尾
- 4-15文字程度
- 例: 「AI時代のnote研究室」「ChatGPT収益化実践研究室」「広告戦略研究室」

### 3-3. キーワード7つ

Phase 2で選定した7つのキーワードをリスト形式で出力。

```
keyword  {キーワード1} {キーワード2} {キーワード3} {キーワード4} {キーワード5} {キーワード6} {キーワード7}
```

### 3-4. 紹介文（Kindle商品ページ用）

#### Kindle紹介文の制約（絶対遵守）

- emoji禁止（✅❌🔥等は使わない）
- 区切り線禁止（---、━━━、■■■等は使わない）
- 特殊記号禁止（★☆●○△▲等は使わない）
- HTMLタグ禁止
- 目次はキーワードSEO対策として必ず含める
- プレーンテキストのみ（太字・斜体もなし）

#### 紹介文の構成テンプレート

原稿のトーンに合わせて以下4タイプから最適なものを自動選択する。

**タイプA: おだやか共感型**（初心者向け・生活改善系）
```
[読者の悩み・不安への共感（2-3文）]

[自分も同じだったという体験談（2-3文）]

[解決策としての本書の紹介（2-3文）]

[本書で得られること（章ごとに1行ずつ）]
第1章：{章タイトル}
第2章：{章タイトル}
...

[特典・ボーナス情報（あれば）]

[CTA: やさしい語りかけ]

【目次】
{目次全文}
```

**タイプB: 成果アピール型**（副業・稼ぐ系）
```
[時代の危機感・読者の焦り（2-3文）]

[疑問を投げかける → 答えはAIだ（2-3文）]

[ステップ形式で本書の内容紹介]
STEP1：{概要}（第N章）
STEP2：{概要}（第N章）
STEP3：{概要}（第N章）

[特典・ボーナス情報（あれば）]

[緊急性・希少性の訴求]

[CTA: 行動を強く促す]

目次
{目次全文}
```

**タイプC: BtoB戦略型**（ビジネス・マーケティング系）
```
[業界の課題・読者のペインポイント（3-4文）]

[自分の失敗体験からの転換（2-3文）]

[本書のコア戦略の説明（2-3文）]

[本書で得られること（箇条書き形式）]
・{得られること1}
・{得られること2}
・{得られること3}
・{得られること4}
・{得られること5}

[特典・ボーナス情報（あれば）]

[CTA: ビジネス的なトーン]

目次
{目次全文}
```

**タイプD: 技術入門型**（IT・テクノロジー・図解マンガ系）
```
[ボリューム・特長のアピール（1-2文）]

[読者が抱える疑問・不安（3-5個の問いかけ）]
「{疑問1}」
「{疑問2}」
「{疑問3}」

[本書ならこれらを解決できるという宣言]

[本書で身につくこと・できるようになること（箇条書き）]
・{できること1}
・{できること2}
・{できること3}

[本書の特長リスト]
・{特長1}
・{特長2}
・{特長3}

[対象読者]
・{対象1}
・{対象2}
・{対象3}

[CTA: 新しい世界への一歩を促す]

【目次】
{目次全文}
```

#### タイプ自動選択の基準

| 原稿の特徴 | 選択タイプ |
|------------|-----------|
| 初心者向け、生活改善、やさしいトーン | タイプA: おだやか共感型 |
| 副業、収益化、稼ぐ、金額目標あり | タイプB: 成果アピール型 |
| BtoB、マーケティング、広告、戦略 | タイプC: BtoB戦略型 |
| IT技術、図解、マンガ付き、ツール解説 | タイプD: 技術入門型 |

複数の特徴が該当する場合は、最も強い特徴に合わせて選択する。

## 使い方

```
# 基本的な使い方
原稿テキストを貼り付けて「紹介文を作って」「Kindleのメタデータを作って」と依頼

# 既にキーワードが決まっている場合
原稿テキスト + 「keyword: {KW1} {KW2} ...」を一緒に渡す
→ ユーザー指定キーワードを優先的に採用

# タイトルだけ再生成したい場合
「タイトルだけ別案を出して」→ Phase 3のタイトル部分のみ再実行

# 紹介文のタイプを指定したい場合
「タイプBで紹介文を作って」→ 指定タイプで生成
```
