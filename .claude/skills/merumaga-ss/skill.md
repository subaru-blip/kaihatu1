# Merumaga SS - メルマガ作成スキル

## Overview

メールマガジン（メルマガ）の**件名**と**本文**をテキスト出力するスキル。
太陽スタイル + 6つの教育要素 + ヘッドライン12パターンを統合し、
**単発メルマガ**から**365日ステップメルマガ**まで一括生成に対応。

## When to Use

```
「メルマガを書いて」
「メールマガジンを作って」
「メルマガの件名と本文を作成して」
「/merumaga-ss」
「配信メールを作りたい」
「365日分のメルマガを作って」
「30日分のメルマガをまとめて生成して」
```

## 動作モード

### モード1: 単発メルマガ
1通のメルマガを生成する。

### モード2: まとめて生成（ステップメルマガ）
指定日数分（7日/14日/30日/90日/365日等）のメルマガをまとめて生成する。
6つの教育要素に基づいてシナリオ設計し、各通の件名+本文を出力。

ユーザーが日数や通数を指定したらモード2で動作する。

## 入力（ユーザーから聞く情報）

以下をユーザーに確認する（わからない項目はスキップ可）：

| 項目 | 説明 | 必須 |
|------|------|------|
| **テーマ/トピック** | メルマガの主題 | YES |
| **目的** | 教育/販売促進/関係構築/告知 | YES |
| **通数/日数** | 単発 or まとめて何通 | YES |
| **ターゲット** | 読者層（初心者/中級者/上級者等） | 推奨 |
| **CTA（行動喚起）** | 読者にしてほしいこと | 推奨 |
| **商品/サービス名** | 販売する場合の商品名 | 販売時必須 |
| **トーン** | カジュアル/フォーマル/親近感/権威的 | 任意 |
| **文字数目安** | 500〜2000字/通（デフォルト: 1000〜1500字） | 任意 |

## 1. 件名（Subject Line）の作成ルール

### 件名パターン 12型（太陽スタイルヘッドライン統合）

| # | パターン | 例 | 心理効果 |
|---|---------|---|---------|
| 1 | **数字+ベネフィット型** | 「たった3日で〇〇が変わった方法」 | 具体性・信憑性 |
| 2 | **疑問+好奇心型** | 「なぜ97%の人が〇〇で失敗するのか？」 | 好奇心喚起 |
| 3 | **警告+恐怖型** | 「【警告】〇〇を続けると...」 | 損失回避 |
| 4 | **秘密+独占型** | 「誰も教えてくれない〇〇の真実」 | 独占欲 |
| 5 | **逆説型** | 「〇〇するほど失敗する理由」 | 認知的不協和 |
| 6 | **ストーリー型** | 「借金300万から月収100万になった話」 | 感情移入 |
| 7 | **緊急+限定型** | 「【本日23:59まで】〇〇」 | 緊急性・希少性 |
| 8 | **共感型** | 「〇〇で悩んでいるあなたへ」 | 自己関連性 |
| 9 | **好奇心ギャップ型** | 「〇〇の裏側、見せます」 | 情報ギャップ |
| 10 | **対比+ビフォーアフター型** | 「月収15万→月収150万。変えたのは〇〇だけ」 | 変化の証明 |
| 11 | **権威+証言型** | 「〇〇の専門家が認めた唯一の方法」 | 権威性 |
| 12 | **挑戦+煽り型** | 「これができないなら〇〇は諦めてください」 | 反発心利用 |

### 件名の鉄則

- **文字数**: 20〜30文字（スマホ表示最適化）
- **パーソナライズ**: 可能なら読者名を含める
- **禁止ワード**: スパム判定される語句を避ける（「無料」「儲かる」の多用等）
- **A/Bテスト用**: 必ず2パターン以上提案する
- **件名に数字を入れると開封率1.5倍**（太陽スタイルデータ）

## 2. 冒頭（Opening）の作成ルール

### 冒頭3パターン

```
A. 共感スタート
   「〇〇って思ったことありませんか？」
   → 読者の悩み・感情に寄り添う

B. 衝撃スタート
   「実は昨日、とんでもないことが起きました。」
   → 好奇心を刺激して読み進めさせる

C. 結論スタート
   「今日は〇〇の方法をお伝えします。」
   → 忙しい読者に配慮、価値を先に提示
```

### 冒頭の鉄則

- **最初の3行で勝負が決まる**
- 挨拶は簡潔に（「清水です。」1行で十分）
- すぐに本題への橋渡しを行う

---

## 3. 本題（Body）の作成ルール

### 目的別テンプレート

#### A. 教育型メルマガ

```
1. テーマの提示
2. なぜ重要なのか（問題提起）
3. 解決策・ノウハウ（3つのポイント等）
4. 具体例・事例
5. まとめ + 実践の促し
```

#### B. 販売促進型メルマガ

```
1. 痛みの共感（恐怖トリガー）
2. 解決策の提示（希望トリガー）
3. 商品/サービスの紹介
4. 実績・お客様の声（社会的証明）
5. オファーの詳細
6. 緊急性・限定性
```

#### C. 関係構築型メルマガ

```
1. 個人的なエピソード
2. そこから得た学び
3. 読者への問いかけ
4. 価値提供（小さなTips）
5. 次回への期待感
```

#### D. 告知型メルマガ

```
1. ニュースの概要（What）
2. なぜ重要か（Why）
3. 詳細情報（When/Where/How）
4. 読者へのメリット
5. アクション案内
```

### 本文の太陽スタイル適用

| 要素 | ルール |
|------|--------|
| 語尾 | 断定55% + 疑問25% + その他20% |
| キラーワード | 500字に3〜5個配置 |
| 心理トリガー | 恐怖→希望サイクルを最低2回 |
| 文長 | 短文48% + 中文32% + 長文20% |
| 段落 | 3〜5行で改行、空白行を挟む |
| 1文1行 | 原則として守る |
| 感情ジェットコースター | 恐怖→希望→恐怖→希望を繰り返す |

## 4. CTA（Call to Action）の作成ルール

### CTAパターン

```
■ リンククリック型
  ▼ 詳しくはこちら
  → [URL]

■ 返信誘導型
  このメールに「〇〇」と返信してください。

■ 購入誘導型
  ▼ 今すぐ手に入れる（残り◯名）
  → [URL]

■ 次回予告型
  明日のメールで〇〇をお伝えします。
  絶対に見逃さないでください。
```

### CTAの鉄則

- **1メール1CTA**（複数あると迷う）
- CTAボタン/リンクは**最低2回**本文中に配置
- 行動しないデメリットも添える

---

## 5. 追伸（P.S.）の作成ルール

### 追伸パターン

```
A. 緊急性の追伸
   「P.S. この特別価格は◯月◯日まで。」

B. 個人的な追伸
   「P.S. 実は僕も最初は〇〇でした...」

C. 次回予告の追伸
   「P.S. 明日は〇〇について、衝撃的な事実をお伝えします。」

D. ボーナスの追伸
   「P.S. このメールの読者限定で〇〇をプレゼント。」
```

## ステップメルマガ一括生成（モード2）

### 6つの教育要素に基づくシナリオ設計

ユーザーが日数を指定したら、以下のフレームワークでシナリオ全体を設計してから各通を生成する。

#### 教育要素マッピング

| 教育要素 | 目的 | 心理トリガー |
|---------|------|------------|
| **1. 目的** | 理想の未来を描く | 希望・快楽トリガー |
| **2. 問題** | 現状の危機認識 | 損失回避・恐怖トリガー |
| **3. 解決策** | 具体的な方法提示 | 権威・専門性トリガー |
| **4. 価値** | 投資対効果を理解 | 対比・数値トリガー |
| **5. 信用** | 信頼の確立 | 社会的証明トリガー |
| **6. 行動** | 具体的行動促進 | 緊急性・希少性トリガー |

#### 日数別シナリオテンプレート

##### 7日間（短期集中）

```
Day 1: 目的 ─ ウェルカム+理想の未来
Day 2: 問題 ─ 現状の危機
Day 3: 問題→解決 ─ なぜ解決できないのか+光
Day 4: 解決策 ─ 具体的な方法
Day 5: 価値+信用 ─ 実績と証拠
Day 6: 行動 ─ オファー開始
Day 7: 行動 ─ 最終案内（締切）
```

##### 14日間（標準）

```
Day 1-2:  目的 ─ 理想の未来+世界観共有
Day 3-4:  問題 ─ 痛みの深掘り
Day 5-6:  解決策 ─ ノウハウ提供
Day 7-8:  価値 ─ 投資対効果
Day 9-10: 信用 ─ 実績・お客様の声
Day 11-12: 行動 ─ オファー開始
Day 13:   行動 ─ 追撃（残りわずか）
Day 14:   行動 ─ 最終締切
```

##### 30日間（充実）

```
Week 1 (Day 1-7):   目的+問題 ─ 世界観構築
Week 2 (Day 8-14):  問題+解決策 ─ 知識提供
Week 3 (Day 15-21): 解決策+価値 ─ 深掘り+証拠
Week 4 (Day 22-28): 信用+行動 ─ 信頼構築→オファー
Day 29-30:          行動 ─ 最終クロージング
```

##### 90日間（長期育成）

```
Month 1 (Day 1-30):  目的+問題 ─ 関係構築+教育
Month 2 (Day 31-60): 解決策+価値 ─ 深い学び+実績
Month 3 (Day 61-90): 信用+行動 ─ 信頼確立→販売
```

##### 365日間（年間ステップ）

```
Q1 (Day 1-90):    基盤構築 ─ 目的+問題（認知・信頼構築）
Q2 (Day 91-180):  価値提供 ─ 解決策+価値（深い教育）
Q3 (Day 181-270): 信頼深化 ─ 信用+事例（ファン化）
Q4 (Day 271-365): 収穫期 ─ 行動+オファー（セールス+リピート）

各Quarterの中で週テーマを設定：
- 週1-2: メインテーマの教育
- 週3: ケーススタディ/事例
- 週4: まとめ+次週予告
```

### まとめて生成時の出力ルール

1. **まずシナリオ全体の設計書を出力**（どの日に何を送るか一覧表）
2. **1通ずつ件名+本文を出力**
3. **各通にフッターを必ず含める**
4. **通し番号をつける**（Day 1, Day 2, ...）
5. **大量生成時はファイル出力も可**（output/merumaga/ に保存）

## 品質チェックリスト

生成後、以下を自動チェック：

- [ ] 件名が20〜30文字以内
- [ ] 件名にキラーワードが含まれている
- [ ] 冒頭3行で引き込みがある
- [ ] 1文1行が守られている
- [ ] 空白行で適切に区切られている
- [ ] CTAが明確で2回以上配置
- [ ] 追伸がある
- [ ] 語尾パターンが偏っていない
- [ ] スマホで読みやすいレイアウト
- [ ] スパム判定ワードを多用していない
- [ ] **フッター（配信元+配信解除）が含まれている**（必須）
- [ ] ステップメルマガの場合、教育要素の流れが適切
