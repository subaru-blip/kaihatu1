## Layer 1: note 競合分析

### 主な競合記事
- AIworker社: SEO型「完全入門ガイド」で上位独占
- クリーヴァ社: セキュリティ警告を前面に出した企業記事
- 体験記型: 「10日間で体験した22のこと」等リスト形式
- 思想型: 「OSの再定義」等の大きな概念で語る記事
- 逆張り型: 「祭りに乗らなかった理由」「限界論」

### 成功パターン
1. 「完全入門ガイド」SEOドミナント型
2. セキュリティ警告・リスク啓発型
3. 思想・ポエム型（有識者オピニオン）
4. 体験記・N日間レポート型
5. 逆張り・限界論型

### 差別化チャンス
- 非エンジニア向けの「結局何ができるの？」実用ガイドが弱い
- OpenClaw + Claude Code + バイブコーディングを横断した記事が不在
- 「じゃあ具体的にどうすればいい？」安全運用ガイドが不足

## Layer 2: ターゲットの悩み・ニーズ

### よくある悩み TOP10
1. 「AIエージェントって結局何ができるの？」概念がつかめない
2. 環境構築で詰む（最初の一歩が踏み出せない）
3. 指示通りに動かない（プロンプトの書き方がわからない）
4. エラー地獄で心が折れる
5. 料金が怖い・見えない（コストへの不安）
6. セキュリティが怖い（OpenClawの権限問題）
7. 教材がすぐ古くなる
8. 丸投げしたら迷走した（設計力の欠如）
9. 何から学べばいいかわからない（学習ロードマップの不在）
10. AIに仕事を奪われるのでは（キャリア不安）

### 初心者がつまづく壁
- 環境構築の壁（ターミナル操作、APIキー取得）
- プロンプト設計の壁（曖昧な指示→的外れな出力）
- コスト・セキュリティの壁（従量課金の恐怖、権限設定）
- 心理的な壁（自分にはできない、学んでも無駄では）

## 総合分析: 記事の方向性

### 推奨する切り口
「非エンジニア向けの"結局OpenClawって何ができるの？"完全ダイジェスト」

既存記事の多くはエンジニア・テック層向け。電子書籍の強みである「パソコン苦手な人でもわかる」切り口をそのまま活かし、10章の内容をギュッと凝縮した入門記事にする。

### 盛り込むべきポイント
1. OpenClawを「24時間AI秘書」と一言で定義（概念の明確化）
2. 3つのスゴさを具体例で紹介（記憶・能動性・PC操作）
3. セキュリティの注意点を簡潔に（怖がらせすぎない）
4. 始め方の概要（詳細は電子書籍に誘導）
5. Claude Code・バイブコーディングにも触れる（トレンド性）
6. 電子書籍への導線を自然に設置
