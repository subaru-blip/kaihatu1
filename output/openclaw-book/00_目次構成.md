---

## 書籍コンセプト

- **ターゲット**: PCに詳しくない一般の方（非エンジニア）
- **文体**: です/ます調で統一。親しみやすく丁寧な語り口
- **スタイル**: 漫画化前提。説明（です/ます調）+ 会話（本文中に自然に組み込み）+ 図解（インフォグラフィック）の3要素構成。原稿に【漫画シーン】やト書きは入れない（漫画は別工程で制作）
- **図解**: 各小見出し（###）ごとに最低1枚の図解タグを挿入。難解箇所は追加。全画像は図解・インフォグラフィックスタイル（漫画風イラストは使用しない）
- **ゴール**: OpenClawが「何なのか」「何がすごいのか」「どう始めるのか」「何に気をつけるのか」を理解できる
- **テーマ**: OpenClaw / Claude Code / バイブコーディング ── 今話題の3つを完全解説
