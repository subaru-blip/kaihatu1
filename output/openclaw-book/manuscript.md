## 1-1. 今までのAIとどう違うの？

「OpenClawって何？」と聞かれたら、こう答えるのが一番分かりやすいです。

> **24時間あなたの代わりに仕事してくれるAI秘書**

「ChatGPTと何が違うの？」と思う方も多いでしょう。ChatGPTは聞いたことに答えるだけですが、OpenClawは**実際にパソコンを操作して仕事をしてくれる**のです。

ChatGPT、Siri、Claudeなど、普段使われているAIには共通点があります。**「こちらから話しかけないと動かない」**という点です。しかも一度会話を終えると、昨日の会話内容も、あなたの仕事のやり方も忘れてしまいます。

OpenClawはここが根本的に違います。以下の図をご覧ください。

![今までのAIとOpenClawの違い](images/ch01_01_ai_comparison.png){ width=100% }

・**動くタイミング**
　・今までのAI：話しかけた時だけ
　・OpenClaw：**24時間、自分から動きます**
・**記憶**
　・今までのAI：会話が終わるとリセット
　・OpenClaw：**全部覚えています**
・**できること**
　・今までのAI：文章を作る、質問に答える
　・OpenClaw：**パソコンを実際に操作します**
・**使う場所**
　・今までのAI：ブラウザやアプリを開く
　・OpenClaw：**普段のチャットアプリ**

たとえば、Slackで「明日のミーティング資料まとめて」と送るだけで、OpenClawはGoogleドライブの議事録と売上データを確認し、資料を作成して報告してくれます。

## 1-2. OpenClawが生まれたストーリー

2024年10月、AI企業のAnthropic（アンソロピック）が**Computer Use**という技術を発表しました。AIにパソコンを操作させる技術です。

これを見たオーストリアのエンジニア、**Peter Steinberger（ピーター・シュタインバーガー）**氏が「Slackから操作できるようにしたら面白い」と思い立ち、**たった1回の週末**で最初のバージョンを完成させました。

![OpenClaw誕生の流れ](images/ch01_02_birth_story.png){ width=100% }

Peter氏はPSPDFKit（ピーエスピーディーエフキット）というPDF関連企業の創業者で、もともとApple開発者コミュニティでは著名な人物でした。その彼が週末に軽い気持ちで作ったプロジェクトが、世界を巻き込む大きなムーブメントになるとは、本人も予想していませんでした。

2025年11月にオープンソースとして公開すると、あっという間に世界中に広がりました。

## 1-3. 名前が3回も変わった！？

OpenClawは1週間で3回名前が変わるという前代未聞の事態を経験しています。

![OpenClawの名前変遷](images/ch01_03_name_history.png){ width=100% }

### 最初の名前：Clawd（クロード）

AI「Claude」の名前と「Claw（爪）」を掛け合わせた造語です。ロブスターのハサミ（claw）にちなんでいました。しかしAnthropicの法務チームから「名前が似すぎている」と連絡が入り、改名を余儀なくされました。

### 2番目の名前：Moltbot（モルトボット）

「Molt」はロブスターの脱皮を意味します。「新しい殻に着替えた」というユーモアを込めた名前でしたが、これもしっくり来ませんでした。

### 最終的な名前：OpenClaw（オープンクロー）

「Open（オープン）」＋「Claw（爪・ハサミ）」。オープンソースであることとロブスターのアイデンティティを両立させた名前に落ち着きました。ロブスターは成長するために何度も殻を脱ぎます。名前が変わったのも、成長の証と言えるかもしれません。

2025年11月     → Clawd 誕生
2026年1月27日  → Moltbot に改名
2026年1月30日  → OpenClaw に改名（最終決定）

## 2-1.「全部覚えてる」～ 昨日の会話も2週間前の話も忘れない ～

Siriに「昨日何を話したか覚えてる？」と聞いても、何も覚えていません。ChatGPTも新しい会話を始めると前回の内容は消えてしまいます。

OpenClawは違います。あなたとの会話を**すべて記憶**しています。先週ポロッと言った「来月引っ越すんだよね」という一言も覚えています。使えば使うほど、仕事のやり方、好み、スケジュールを理解していきます。

![記憶の違い](images/ch02_01_memory_before_after.png){ width=100% }

たとえば「先週話した企画書の件、続きをお願い」と送った場合、ChatGPTは「先週の会話を確認できません」と返しますが、OpenClawは「先週の打ち合わせで決まった方針に基づいて更新しますね。前回の『グラフをもっと入れて』というフィードバックも反映します」と返してくれます。

### なぜ覚えていられるのか

OpenClawはあなたのパソコン（またはサーバー）の中に**会話ログとメモ**を保存しています。クラウド上ではなく手元にデータがあるため、いつでも読み返して思い出すことができる仕組みです。

![OpenClawの記憶の仕組み](images/ch02_02_memory_flow.png){ width=100% }

## 2-2.「向こうから連絡してくる」～ 待つAIから動くAIへ ～

今までのAIは、アプリを開いて話しかけるまでじっと待っているだけでした。

OpenClawは**自分から連絡してきます**。

> OpenClaw: 「緊急のメールが届いています。〇〇社から、明日の会議の時間変更の連絡です」

> OpenClaw: 「先月頼まれたレポートの締め切りが明後日です。まだ未完成のようですが、手伝いましょうか？」

![待つAIと動くAIの違い](images/ch02_03_proactive_before_after.png){ width=100% }

頼まれていなくても、状況に応じて**自分で判断して行動する**のがOpenClawの大きな特徴です。

毎朝こんなメッセージが届くように設定することもできます。

> OpenClaw: 「おはようございます。今日の予定は3件です。10:00 〇〇社ミーティング（資料は昨晩準備済み）/ 14:00 チームランチ（予約確認済み）/ 16:00 週次レポート提出（下書きを作っておきました）。天気は午後から雨です。傘をお忘れなく」

ChatGPTに「メールを送って」と頼んでも、文章を作ってくれるだけで実際にメールを送ることはできません。コピー&ペーストして送るのは自分自身です。

「ブログの新しい記事を書いて、お問い合わせフォームのメール3通に返信しといて」と送れば、OpenClawがすべて処理してくれます。その間、あなたは好きなことをしていられます。

## 3-1. 3つの層で動いている

OpenClawの仕組みは**3つのパーツ**で構成されています。会社に例えると分かりやすくなります。

![3つの層の詳細](images/ch03_01_3layer_detail.png){ width=100% }


### ① 受付（チャットアプリ）

OpenClawに話しかける**入り口**です。Slack、Discord、Telegram、WhatsApp、LINEなど22種類以上のチャットアプリがそのまま受付として使えます。新しいアプリをインストールする必要はありません。

![対応チャットアプリ](images/ch03_02_chat_apps.png){ width=100% }

### ② 司令塔（Gateway＝ゲートウェイ）

受付から届いた指示を受け取り、「何をすればいいか」を判断する頭脳です。あなたのパソコンの中で動いており、以下を管理しています。

![Gatewayの5つの役割](images/ch03_03_gateway_roles.png){ width=100% }

・どのAI（LLM）を使うか
・どんな操作を許可するか
・会話の記録を保存する
・危険な操作をブロックする

大事なのは、この司令塔がクラウドではなく**あなたのパソコンの中にいる**ということです。

### ③ 作業員（LLM＝AI頭脳）

実際に「考える」部分です。Claude、ChatGPT、あるいは無料のローカルAIなど、**好きなAIを選べる**のが大きな特徴です。

たとえば「メール返信して」と指示した場合の流れは次のようになります。

![メール処理の流れ](images/ch03_04_email_flow.png){ width=100% }

---

\newpage

## 3-2.「オープンソース」ってなに？

**オープンソース**とは、設計図を全部公開して、誰でも無料で使えるようにしたソフトウェアのことです。

料理に例えると分かりやすくなります。

![オープンソースとは](images/ch03_05_opensource_before_after.png){ width=100% }

・**普通のソフト** = レストランの料理。完成品を買います。レシピは秘密です。
・**オープンソース** = レシピ公開の料理。誰でも作れますし、アレンジも自由です。

![オープンソースのメリット](images/ch03_06_opensource_merits.png){ width=100% }

**OpenClawがオープンソースであるメリット：**

1. **無料で使えます**（AIの利用料は別途かかる場合があります）
2. **中身が見えます**（怪しいことをしていないか確認できます）
3. **世界中の人が改良できます**（バグが見つかればすぐ修正されます）
4. **自分好みにカスタマイズできます**

---

\newpage

## 3-3.「LLM」ってなに？

**LLM（エルエルエム）**は「Large Language Model」の略です。日本語にすると「大規模言語モデル」になります。

一言で言えば、**「めちゃくちゃ賢いAIの脳みそ」**です。

![LLMの種類と比較](images/ch03_07_llm_comparison.png){ width=100% }

・**Claude**：Anthropic（長い文章が得意、安全性重視）
・**ChatGPT（GPT-4o）**：OpenAI（万能型、世界で最も有名）
・**Gemini**：Google（Google連携が強い）
・**DeepSeek**：DeepSeek社（中国発、コスパが良い）

OpenClawの面白いところは、この「脳みそ」を**自分で選べる**ということです。さらに**ローカルLLM**と呼ばれる、自分のパソコンの中だけで動くAIを選ぶこともできます。この場合、完全に無料で、データが外に出ません。ただしClaudeやChatGPTに比べると性能は少し落ちます。

---

\newpage

## 3-4.「API」ってなに？

OpenClawの設定で必ず出てくるのが**API（エーピーアイ）**という言葉です。

**API**とは、ソフトウェア同士が会話するための**「電話番号」**のようなものです。OpenClaw（司令塔）がClaude（AI頭脳）に仕事を頼む時の連絡先にあたります。

![APIの仕組み](images/ch03_08_api_flow.png){ width=100% }

電話をかけるには電話番号が必要なように、AIを使うには**APIキー**が必要になります。APIキーとは、あなた専用の「会員番号」のようなもので、正式な利用者であることを証明するコードです。

**重要：APIキーは銀行の暗証番号と同じレベルで管理してください。** これが漏れると、あなたのお金で勝手にAIが使われてしまいます。APIキーは使いたいAIの公式サイト（ClaudeならAnthropicのサイト）でアカウントを作って発行します。

---

\newpage

## 3-5.「エージェント」ってなに？

OpenClawを語る上で欠かせない言葉が**「エージェント」**です。

**エージェント**とは、**自分で判断して行動できるAI**のことです。

![普通のAIとエージェントAIの違い](images/ch03_09_agent_comparison.png){ width=100% }

・**行動**
　・今までのAI：言われたことに答えるだけ
　・エージェントAI（OpenClaw）：自分で判断して行動します
・**道具**
　・今までのAI：持っていない
　・エージェントAI（OpenClaw）：パソコン操作という「手」を持っています
・**計画**
　・今までのAI：1回の質問に1回の回答
　・エージェントAI（OpenClaw）：複数のステップを自分で計画して実行します

たとえば「旅行の予約をして」と頼んだ場合を考えてみましょう。今までのAIは「こちらのサイトで予約できますよ」と教えてくれるだけです。一方エージェントAIは、自分でサイトを開き、日程を入れ、料金を比較し、「このプランがおすすめですが、予約しますか？」と聞いてきます。

![旅行エージェントの例](images/ch03_10_agent_travel_flow.png){ width=100% }

## ～ セキュリティ編 ～

![セキュリティ5箇条](images/ch04_header_security.png){ width=100% }

---

OpenClawは便利ですが、使い方を間違えると本当に危険です。知らないまま使うのは、車の運転を知らないまま高速に乗るようなもの。怖がらせたいわけではありませんが、**対策すれば安全に使えます**。この章でしっかり学んでおきましょう。

\newpage

## 4-1. OpenClawは「新入社員に会社の鍵を全部渡す」のと同じ

OpenClawを自分のパソコンで動かすと、以下のすべてにアクセスできる状態になります。

・メール
・ファイル
・ブラウザに保存されたパスワード
・ログイン中のサービス（銀行、SNS、ショッピングサイト）
・パソコン上のすべてのデータ

![OpenClawの権限](images/ch04_01_permission_concept.png){ width=100% }

家政婦さんに鍵を渡すイメージです。リビングやキッチンの鍵はOKですが、金庫の鍵や銀行の通帳はNGです。必要な部屋の鍵だけ渡せばいいのです。

OpenClawの公式もこうアドバイスしています。

> 「この権限を、入社1日目の新入社員に渡しても大丈夫か？答えがNoなら、OpenClawにも与えるべきではありません」

必要な権限だけを与える。これが「**最小権限の原則**」と呼ばれる考え方です。

## 4-2. 実際に起きたセキュリティ事件

OpenClawはまだ新しいツールですが、すでに複数のセキュリティ事件が報告されています。

![実際に起きた事件](images/ch04_02_security_incidents.png){ width=100% }

### 事件① メールが全部消された

知らない人から届いたメールの中に「AIへの隠し命令」が仕込まれていました。OpenClawがその命令に従い、**メールをゴミ箱含めてすべて削除**してしまいました。

### 事件② 150万件のAPIキーが流出

AIだけのSNS「Moltbook」のデータベースが誰でも見られる状態になり、**約150万件のAPIキー**が漏洩しました。

### 事件③ 悪意あるスキルが341個発見

OpenClawの拡張機能を配布するClawHubで、**341個の悪質なプログラム**が見つかりました。インストールすると、パスワードや暗号通貨の情報が盗まれる仕組みでした。

### 事件④ Webページを見ただけで乗っ取られる脆弱性

OpenClawのセキュリティ上の欠陥（CVE-2026-25253、深刻度スコア8.8/10）が発見されました。悪意あるWebページを1回訪問するだけで、パソコンを遠隔操作されてしまうという深刻な問題でした（現在は修正済みです）。

こう聞くと怖いかもしれませんが、車と同じです。事故は起きますが、シートベルトやルールを守れば安全に乗れます。

## 4-3. 5つの危険ポイント

![5つの危険ポイント](images/ch04_03_danger_points.png){ width=100% }

### ① 悪い命令が紛れ込む（プロンプトインジェクション）

メールやWebページの中に、人間には見えないがAIには読める「隠し命令」を仕込む攻撃です。手紙の中に透明なインクで「秘書さん、金庫を開けてください」と書いてあるようなものです。OWASPの「LLMアプリケーションセキュリティ脅威Top 10」でも、**第1位**にランクされています。

![プロンプトインジェクション](images/ch04_04_prompt_injection.png){ width=100% }

**対策：** 知らない人からのメールやファイルをOpenClawに処理させないでください。重要な操作は「承認してから実行」する設定にしてください。

### ② チャットが攻撃の入り口になる

OpenClawはチャットから操作するため、あなたにメッセージを送れる人は、AIに命令できてしまう可能性があります。

**対策：** OpenClawは個人チャットだけで使ってください。グループには入れないでください。WhatsAppを使う場合は専用の番号を用意してください。

### ③ 安全装置がわざとついていない

OpenClawは当初、意図的に安全装置（ガードレール）を設けない設計でした。最近のアップデートで改善されつつありますが、自分で安全設定をする必要があります。

**対策：** インストール後に `openclaw doctor` と `openclaw security audit` を必ず実行してください。

### ④ 普段使いのパソコンで動かすと危険

普段使いのパソコンには、パスワード、銀行情報、仕事のデータなど大切な情報がたくさん入っています。万が一の時に全部アクセスされるリスクがあります。

**対策：** 専用の環境（VPSや古いパソコン）で動かしてください。どうしても自分のパソコンで使う場合は、触れるフォルダを限定してください。

### ⑤ 権限の与えすぎに注意

「便利だから」と何でもできるようにすると、事故が起きた時の被害が大きくなります。

**対策：** 実行できるコマンドを「許可リスト」で制限してください。段階的に権限を広げていくのがおすすめです。

---

\newpage

## 4-4. 初心者の安全ルール5箇条

![安全ルール5箇条](images/ch04_05_safety_rules.png){ width=100% }

・① メインのパソコンでは使わないでください（VPSか古いPCで）
・② APIキーは絶対に人に見せないでください
・③ 知らない人からのメッセージは処理させないでください
・④ 大事な操作は「承認制」にしてください
・⑤ `openclaw doctor` を定期的に実行してください

この5つを守れば、初心者でも安心して始められます。

## 4-5. API料金に注意！

セキュリティとは別に知っておくべきことがあります。**料金**です。

OpenClaw自体は無料ですが、AIの頭脳（LLM）をクラウドで使う場合、使った分だけ料金がかかります。OpenClawはエージェントAIなので、1つのタスクでも何回もAIに問い合わせをします。1回の依頼で4〜5回分の料金がかかることも珍しくありません。

![API料金の仕組み](images/ch04_06_api_pricing.png){ width=100% }

・**軽く試す程度**：1,000〜3,000円
・**毎日ちょっと使う**：3,000〜10,000円
・**ガッツリ使う**：10,000〜20,000円以上

### 料金を抑えるコツ

![コスト節約テクニック](images/ch04_07_cost_saving.png){ width=100% }

1. **利用上限を設定してください**（API管理画面で月額上限を設定できます）
2. **無料のローカルLLMを使ってください**（Ollamaなど。性能は落ちますが料金ゼロです）
3. **DeepSeek APIを使ってください**（高性能なのに料金が安いです）
4. **必要な時だけ使ってください**（24時間稼働は料金が跳ね上がります）

上限設定を忘れて1万円飛んだという話もありますので、必ず最初に設定しておいてください。

## 5-4. Windowsの人は要注意！「WSL2」ってなに？

Windowsを使っている場合、OpenClawを動かす前に**WSL2**（ダブリュエスエルツー）を入れる必要があります。

### WSL2とは

**WSL2 = Windows Subsystem for Linux 2**

Windowsの中にLinux（別のOS）を入れる仕組みです。大変そうに見えますが、**コマンドを1つ打つだけ**でインストールできます。

![WSL2とは](images/ch05_05_wsl2_concept.png){ width=100% }

### なぜWSL2が必要なのか

OpenClawは元々Mac/Linux向けに作られています。Windowsでそのまま動かすと様々な問題が起きるため、「Windowsの中にLinux環境を作り、その中でOpenClawを動かす」方法が推奨されています。

日本のコミュニティでも「WSL2で動かしたら問題なく動いた」という報告が多く、現時点ではこれが最も安定した方法です。次の章で具体的な手順を全部書いていますので、10分もあれば完了します。

## 6-1. ステップ①：Node.jsを入れよう

**Node.js（ノードジェイエス）**は、OpenClawを動かすための土台となるソフトウェアです。Node.jsは「電気」、OpenClawは「家電」に相当します。家電を動かすには電気が必要なように、OpenClawにはNode.jsが必要です。

![Node.jsとOpenClawの関係](images/ch06_02_nodejs_formula.png){ width=100% }

### Macの場合

ターミナルを開いて以下を入力してください。（Finder →「アプリケーション」→「ユーティリティ」→「ターミナル」）

node -v

「v22.x.x」のように表示されたら、もう入っています。次のステップへ進んでください。入っていない場合は、Node.js公式サイトからダウンロードしてインストールしてください。

### Windowsの場合（WSL2が必要）

まずWSL2を入れます。

![Windows WSL2セットアップ手順](images/ch06_03_wsl2_setup.png){ width=100% }

**手順1：** 「PowerShell」を**管理者として**開いてください（スタートボタンを右クリック →「ターミナル（管理者）」）

**手順2：** 以下を入力してEnterを押してください
wsl --install

**手順3：** パソコンを再起動してください

**手順4：** 再起動後、Ubuntuが自動で開くのでユーザー名とパスワードを設定してください

**手順5：** Node.jsを入れてください（Ubuntu内で実行します）
sudo apt update && sudo apt upgrade -y
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
node -v

「v22.x.x」と表示されたら成功です。

「コマンドを入力」と言われてもピンと来ない方もいるかもしれません。パソコンに文字で指示を出す方法のことです。見た目は難しそうに見えますが、書いてある通りにコピー&ペーストするだけなので安心してください。

## 9-4. バイブコーディングとは？

**バイブコーディング（Vibe Coding）**とは、プログラミングの知識がなくても、AIに指示するだけでソフトウェアを作る方法のことです。

OpenAIの共同創業者**アンドレイ・カルパシー**氏が名付けた言葉で、「コードを書くのではなく、**雰囲気（バイブ）を伝えるだけ**でプログラムができる」という意味が込められています。

### 従来のプログラミング vs バイブコーディング

![従来のプログラミング vs バイブコーディング](images/ch09_04_vibe_coding.png){ width=100% }

・**テストします**：AIが修正して完成します

プログラミングの勉強が不要で、日本語が話せれば誰でもできるのがポイントです。

たとえば「体重を毎日記録できるWebアプリを作って。折れ線グラフで推移が見られるように。デザインはシンプルでかわいい感じ」と伝えると、3分後には完成しています。外注したら10万円はかかるものが、3分です。

## Q8. MCP（エムシーピー）って何ですか？

**MCP = Model Context Protocol**。AIと外部ツールをつなぐ「共通規格」です。

![MCP登場前と後](images/ch10_08_mcp.png){ width=100% }

USB-Cをイメージすると分かりやすくなります。USB-Cはどんなスマホでも同じケーブルで充電できる共通規格です。MCPはどんなAIでも同じ方法で外部ツールと接続できる共通規格です。

MCPが登場する前は、AIとGoogleカレンダーをつなぐ専用プログラム、AIとSlackをつなぐ別の専用プログラム…とバラバラでした。MCPにより、1つの共通規格で何でもつなげるようになりました。

OpenClawがGoogleカレンダー、Gmail、Notion、Slackなど様々なサービスと簡単に連携できるのは、MCPのおかげです。
